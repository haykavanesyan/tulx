# Changesets

Этот проект использует [Changesets](https://github.com/changesets/changesets) для управления версиями и changelog.

## Как это работает

### Создание changeset

Когда вы вносите изменения, которые требуют обновления версии, создайте changeset:

```bash
pnpm changeset
```

Эта команда:
1. Покажет список измененных пакетов
2. Попросит выбрать тип изменения (major, minor, patch)
3. Попросит описать изменения
4. Создаст файл в `.changeset/` с описанием изменений

### Обновление версий

После того, как changesets готовы, обновите версии:

```bash
pnpm version
```

Эта команда:
1. Обновит версии пакетов согласно changesets
2. Создаст/обновит CHANGELOG.md для каждого пакета
3. Обновит зависимости между пакетами
4. Удалит использованные changesets

### Проверка статуса

Проверьте, есть ли незавершенные changesets:

```bash
pnpm changeset:status
```

## Типы изменений

- **major** - Breaking changes (несовместимые изменения API)
- **minor** - Новые функции (обратно совместимые)
- **patch** - Исправления багов (обратно совместимые)

## Пример workflow

1. Внесите изменения в код
2. Создайте changeset: `pnpm changeset`
3. Зафиксируйте changeset в git
4. Когда готовы к релизу: `pnpm version`
5. Проверьте изменения и зафиксируйте обновления версий
6. Создайте тег и релиз: `git tag v1.0.0 && git push --tags`
